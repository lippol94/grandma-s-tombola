Detailed Description

This contains the prototypes and data structures for main server application.

Main server thread waits for new clients to connect, arrange them in rooms according to their requestes and starts new games' threads running logic described in gameplay_lib.h

Author
Leonardo Arcari

Feb 2016

Callback function to handle GSocketService's incoming signal.

The signal is sent when a new client connects to the server. A new _player struct is allocated and fields are filled. Related GSocket object is registered a source to check on G_IO_IN condition in server's main event loop.

void lobby_actions ( loop_data_t *  loop_data,
socket_message_t *  socket_message,
player_t *  player 
)

Function to be invoked by read_socket to answer a request from client.

After evaluating message string a proper choice is taken:

  • If a create-game message, a new game room is created and player is added to it if requested game's name doesn't exists. If already exists a call to _lobby_join_game is done to add player to it
  • If a join-game message, player is added to requested game room. If the room doesn't exists a negative answer is sent through lobby_new_game_answer
  • If a start-game message a call to _lobby_start_game is done.
Parameters
loop_dataPointer to a _loop_data struct
socket_messagePointer to a _socket_message struct where _socket_message::msg field contains the string to be evaluated
playerPointer to _player struct of the one who sent the message in first place.

gboolean read_socket ( GSocket *  socket,
GIOCondition  condition,
gpointer  user_data 
)

Callback function to be invoked when G_IO_IN condition is satisfied on a socket.

Message is read from socket, normalized and sent to lobby_actions for evaluation. In case reading funtion from socket returns 0 (which means the client disconnected) housekeeping is done on that player and in case there are no more players, game thread exits.

Returns
TRUE if reading went fine. FALSE otherwise.
